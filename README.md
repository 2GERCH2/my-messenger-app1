# botellón — приватный мессенджер

Полноценное веб-приложение-мессенджер в стиле Telegram Dark Theme с упором на приватность и простоту. Реализован полный цикл регистрации, входа с 2FA, поиска пользователей, списка чатов и живого обмена сообщениями.

## ✨ Особенности

- **Тёмная тема** в стиле Telegram (#0f0f23 фон, #1a1a2e панели)
- **Регистрация с 2FA** — верификация email 6-значным кодом
- **Вход с 2FA** — отправка кода на email при каждом входе
- **Поиск пользователей** с debounce (300мс) по @username
- **Список чатов** — сортировка по последнему сообщению, индикаторы непрочитанных
- **Живой чат** — сообщения в виде «пузырей», статусы отправлено/прочитано
- **Socket.IO симуляция** — real-time события (сообщения, онлайн-статус)
- **Защита от пересылки** — `user-select: none` на сообщениях, нет кнопки «переслать»
- **Модульная архитектура** — Zustand store, сервисы, компоненты, контексты
- **Адаптивный дизайн** — mobile-first, плавные анимации Framer Motion

## 🛠 Технологический стек

**Frontend** (в одном файле `src/App.tsx` для демо):
- React 19 + TypeScript
- Zustand (глобальное состояние)
- Framer Motion (анимации)
- Lucide React (иконки)
- Tailwind CSS v4 (стилизация)
- Mock API + LocalStorage (имитация backend)

**Архитектура (готова к расширению):**
- `services/` — бизнес-логика (auth, chat, search)
- `components/` — переиспользуемые UI-элементы
- `context/` — Toast, Socket контексты
- `hooks/` — кастомные хуки (useDebounce)
- Socket события разделены по модулям

## 📱 Экраны приложения

### 1. Регистрация (`/register`)
- Поле @username с автодобавлением «@» и проверкой уникальности в реальном времени
- Email для двухфакторной аутентификации
- Пароль с индикатором надёжности (слабый/средний/сильный)
- Модальное окно ввода 6-значного кода верификации

### 2. Вход (`/login`)
- Поля @username и пароль
- После успешной проверки — модальное окно ввода 2FA кода
- Таймер 60с для повторной отправки кода

### 3. Список чатов (`/chats`)
- Верхняя панель с логотипом botellón и аватаром
- Поиск пользователей по @username с выпадающими результатами
- Список диалогов с последним сообщением и временем
- Боковое меню (бургер) с настройками и выходом

### 4. Чат с пользователем (`/chat/:userId`)
- «Пузырьковая» лента сообщений (свои — справа, чужие — слева)
- Статусы сообщений: ✓ отправлено, ✓✓ прочитано
- Поле ввода с auto-resize, отправка по Enter (Shift+Enter — перенос строки)
- Сообщения защищены от выделения и пересылки

## 🔐 Безопасность

- **Хеширование паролей** — простая демонстрационная функция (в продакшене заменить на bcrypt)
- **2FA на каждый вход** — повышает безопасность учётной записи
- **Изоляция данных** — каждый пользователь видит только свои чаты
- **Запрет пересылки** — на уровне UI и CSS (`user-select: none`)
- **Валидация ввода** — на клиенте и (заглушка для) сервере

## 🚧 Ограничения демо-версии

- Backend эмулируется через LocalStorage (нет настоящего Node.js сервера)
- Email отправка — заглушка (код показывается в интерфейсе)
- Нет загрузки файлов/изображений
- Нет групповых чатов
- Нет end-to-end шифрования

## 🔮 Планы по расширению (архитектура готова)

1. **Подключение настоящего backend** (Node.js + Express + Socket.IO)
2. **Prisma + PostgreSQL** для хранения данных
3. **Nodemailer** для реальной отправки 2FA кодов
4. **Загрузка и отправка файлов/изображений**
5. **Групповые чаты** (модель Chat с множеством участников)
6. **Push-уведомления** (Web Push API)
7. **End-to-end шифрование** (libsignal или аналог)
